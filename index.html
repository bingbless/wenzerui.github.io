方法

定义

类似于其它语言的函数，是一段用来完成特定功能的代码片段，一般情况是，定义一个方法包含以下语法

方法包含一个方法头和方法体，以下是一个方法的所有部分：

  1. 修饰符：可选，告诉编译器如何调用该方法，定义了该方法的访问类型。
  2. 返回值类型：是方法返回值的数据类型，有些方法没有返回值，返回值类型为void
  3. 方法名：方法的实际名称
  4. 参数类型和参数名：当方法被调用时，传递值给参数。这个值被称为实参或者变量。参数列表是指方法的参数类型、顺序和参数的个数。参数是可选的，方法可以不包含任何参数。
  5. 方法体：具体的语句，定义该方法的功能。

形参和实参

实际参数：实际调用传递给他的参数

形式参数: 用来定义作用的

java在方法内复制了一份变量，所以在方法内的改变不会影响方法外的变量

调用

调用方法：对象名.方法名(实参列表)

两种调用的方式：

当返回值为一个值时，方法调用通常会被当做一个值

如果返回值是void，方法调用一定是一条语句

值传递和引用传递

值传递(pass by value)：在调用函数时，将实际参数复制一份传递到函数中，这样在函数中对参数进行修改，就不会影响到原来的实际参数；

**引用传递(pass by reference):**在调用函数时，将实际参数的地址直接传递到函数中。这样在函数中对参数进行的修改，就会影响到实际参数；

Java中的确没有引用传递，当把一个引用类型的变量作为参数传递给方法的时候，也仅仅是将引用变量的值复制一份给了方法中的参数，自己始终指向原本的对象。

重载

定义：重载就是在一个类中，有相同的函数名称，但形参不同的函数

规则：

  1. 方法名称必须相同
  2. 参数列表必须不同（个数不同、或类型不同、参数排列不同等）
  3. 方法的返回类型可以相同也可以不相同
  4. 仅仅返回类型不容不足以成为方法的重载

可变参数

在方法声明中，在指定参数类型后加一个…

一个方法只能指定一个可变参数，它必须是方法的最后一个参数。

变量

可以变化的量

Java是一种强类型语言，每个变量都必须声明其类型

类变量 static

从属于类（目前个人理解是一个类不管创建多少个对象都共用这一个类变量）

实例变量

在类里面方法外面声明

从属于对象

如果不自行初始化这个类型的默认值

  1. 布尔值默认为false
  2. 除了基本变量都为null

局部变量

在方法里声明的变量，作用域仅是该方法

必须声明和初始化值！！！

变量的命名规则

  1. 原则：见名知意
  2. 类成员变量、局部变量和方法名：首字母小写和驼峰原则：monthSalary
  3. 常量：大写字母和下划线：MAX_VALUE
  4. 类名：首字母大写和驼峰原则：Man, GoodMan

自动类型转换

一些小知识

数字之间可以用_分割，输出时不输出下划线

在计算中，会先计算结果，再进行数据类型转换，这样会导致溢出，所以应该在计算中就进行数据类型转换

Note

  1. 不能对布尔值进行转换
  2. 不能把对象类型转化为不相干的类型
  3. 在把高容量转化为低容量时，强制转换
  4. 转换的时候可能存在内存溢出，或者精度问题（强制转化丢失精度，不四舍五入）
